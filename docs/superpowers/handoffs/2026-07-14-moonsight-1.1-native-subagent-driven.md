## 一句话目标

交付 **MoonSight 1.1 Linux x86_64 全原生桌面 Host**：

- 运行时 **无** WebView / JavaScript / DOM / wasm
- 与 **Tauri 正式对等**（证据全 PASS 前不得声称 coequal）
- 与 Web/Tauri **共用** 项目源、MSB、manifest、资源、catalogs、UI 包、save schema
- **默认** `moonsightc build` 仍是 **Web**；native 必须显式 `--backend native-linux-x86_64`
- 栈已选定且已批：**SDL3 3.4.12 + wgpu-native 29.0.1.1 + FreeType 2.14.3 + miniaudio 0.11.25**
- 集成基线 = **当前 `main`**（Formal 1.0 **产品面已合入**；1.0 外部 W1/D1/C1 仍可能 BLOCKED——与 1.1 实现并行，**不要**把「修 1.0 证据矩阵」当成本任务的一部分）

## 硬门禁（违反即停）

1. **Task 2 垂直切片未 PASS → 禁止 Task 3+**（禁止 `host_core` 抽取、禁止改 `host_web` 生产路径）。
2. **换栈**（Sokol / Dawn / SDL_GPU）仅在 Task 2 确认 PRD 六类**架构不可行**之一，并完成新的 Planner→Architect→Critic 修订后；实现 agent **不得**自行换栈。
3. **单活跃 `HostSession`**：进程全局 mixer 同一时刻只能被一个会话占用。
4. **C 回调永不调用 MoonBit**；FFI 非原始参数必须 `#borrow` / `#owned`。
5. **Formal 1.0 W1/D1/C1 只读**；1.1 用独立证据 ID（`N1-WAYLAND-MESA`、`N1-X11-NVIDIA`、`N1-GLIBC235-SMOKE` 等）。
6. 压缩包 **≤ 150 MiB** 除非完整共识修订。
7. 生产路径 **禁止** 静默回落 demo / 静默换后端 / 空渲染器继续。
8. 存档：**schema 共享（写 v5 / 读 v2–v5），路径不互通**；不做 Web/Tauri/native 自动迁移。

## 架构摘要（实现时对照）

```
host_web (wasm 薄适配，export 名不变)
    └── host_core (HostSession，目标中立)
host_native (MoonBit main + gated extern "c")
    └── ms_native_* C ABI
            ├── window_sdl.c
            ├── gpu_wgpu.c      (+ render/shaders/*.wgsl 同源)
            ├── text_freetype.c
            ├── audio_miniaudio.c
            └── persistence_posix.c  (XDG + .tmp/.bak)
```

- Web：现有 float DrawList pack  
- Native：`NativeFramePacket` v1，`MSFR` 头 + 同 stride  
- 字体：固定 Noto Latin + CJK digest；Web 双字体迁移是**显式共享合同**，不是 native 副作用  
- ELF：Ubuntu 22.04 构建，**≤ GLIBC_2.35**，RUNPATH `$ORIGIN/../lib`，捆绑 `libwgpu_native.so`

## Subagent-Driven 操作规程（主代理必做）

### 每个 Task 开始前

1. 读 plan 中该 Task 的 **Files / Steps** 全文。
2. 读 design 中对应章节（见 plan 文末 Spec coverage 表）。
3. `git status` / 当前 branch / worktree 路径确认无误。
4. 用 `using-git-worktrees`（若尚未隔离）确保在 1.1 worktree 内。
5. 写给子代理的 prompt 必须包含：
   - 只做 **Task N**，做完即停
   - 精确文件列表与完成定义
   - TDD：先失败测试 → 实现 → 绿 → 按约定 message commit
   - 禁止事项（硬门禁列表摘要）
   - 验证命令与 Expected
   - 要求回报：改了哪些文件、测试输出摘要、未决风险

### 每个 Task 结束后（主代理审查清单）

- [ ] plan 中该 Task 的 checkbox 步骤是否都有对应 diff/证据
- [ ] 测试命令真实跑过且 PASS（不是「应该会过」）
- [ ] 无占位/假实现（TODO、skip test、空 stub 冒充完成）
- [ ] Web 默认路径与 Formal 1.0 证据脚本未被破坏（相关回归仍绿）
- [ ] commit message 符合计划
- [ ] 若是 Task 2：垂直切片证据文档已写；ASan/ELF/glibc 有记录；**未**提前改 host_web 生产逻辑
- [ ] 若是 Task 3+：Task 2 PASS 记录仍在

审查失败 → 同一 Task 开修复子代理，**不要**进入下一 Task。

### 子代理类型建议

| 角色 | 建议 |
|------|------|
| 实现 | `executor` / general-purpose（复杂 Task 用更强模型） |
| 调查 | `explore` / `tracer`（仅当阻塞） |
