## 1. 背景与目标

### 1.1 问题

Formal 1.0 产品面（save v5、locale catalogs、rollback、Web/Tauri 打包与发布工具）已在 `main`。桌面路径仍依赖 **Tauri + WebView + wasm + JS Host**。战略上需要真正的原生桌面运行时：不依赖浏览器技术，同时保持与 Web/Tauri 同一套作者/内容契约。

已有 RALPLAN（PRD + 实现计划 + test-spec）经 Architect iteration 5 与 Critic iteration 2 **APPROVE**，但材料位于 `.omx/`，且同步假设仍写「Formal 1.0 主要在独立 worktree」。本设计将批准结论收口到 durable superpowers 路径，并修正集成基线。

### 1.2 一句话目标

交付 **生产级 Linux x86_64 原生桌面后端**，与 Tauri **正式对等**；运行时无 WebView / JavaScript / DOM / wasm；同一项目源、MSB、manifest、资源、catalogs、UI 包与 save schema 可被 Web/Tauri 与 native 消费。

### 1.3 产品原则

1. **一个引擎，多个 Host** — 叙事、UI、locale、save、rollback、音频意图、输入意图、DrawList 语义归 MoonBit。
2. **对等先于新特性** — 1.1 不为 native 增加新叙事/演出功能；先复现 Formal 1.0 桌面可见行为。
3. **不拿回归换进度** — 不得削弱默认 Web 构建、Tauri 打包、Web 包校验、Formal 1.0 证据矩阵。
4. **项目自有边界** — MoonBit 只见版本化 `ms_native_*` C ABI；vendor 结构体不进入 MoonBit。
5. **支持靠证据** — 自动化与真机门禁全 `PASS` 前，不得声称 native 与 Tauri coequal。

### 1.4 范围

#### 必做

| 区域 | 内容 |
|------|------|
| 共享 Host | 从 `host_web` 抽出 target-neutral `host_core`；**进程内恰好一个活跃 `HostSession`** |
| Native 栈 | SDL3 窗口/输入；wgpu-native + 规范 WGSL；FreeType 字图；miniaudio |
| 持久化 | XDG + Tauri 等价原子写/恢复；JSON 与 save **写 v5 / 读 v2–v5** |
| 工具链 | 显式 `--backend native-linux-x86_64`；**Web 仍为默认** |
| 交付 | `tar.gz` 玩家包 + ELF/glibc 门禁 + **独立** 1.1 证据体系 |
| 字体 | 固定 Noto Latin + CJK 双字体栈（Web/native 同序同 digest） |
| 符合性 | Web↔native 规范化 trace；canonical demo 完整可玩 |

#### 明确不做

- Windows / macOS 正式支持声明或安装器
- Web ↔ Tauri ↔ native 存档路径自动迁移
- AppImage / Flatpak / 发行版仓库 / 商店 / 自动更新
- HarfBuzz 复杂文种 shaping 合同变更（评估保留，1.1 不链接）
- Live2D / 粒子 / 时间轴等新演出系统
- **修改 Formal 1.0 的 W1/D1/C1 证据矩阵或候选语义**
- 并发多 `HostSession` / 将 Mixer 所有权迁入 `Engine`（列为未来共享运行时变更）

### 1.5 相对已批 RALPLAN 的同步修订

| 原假设 | 本设计 |
|--------|--------|
| Formal 1.0 产品主要在 `.worktrees/formal-1.0-release` 并行 | **产品面已合入 `main`**；1.0 **外部 W1/D1/C1 证据仍 BLOCKED** |
| 与 1.0 worktree 双头同步 | **集成基线 = 当前 `main`**；1.1 在 **独立 worktree/分支** 实现 |
| 共享文件冲突主要在 release tooling | 仍避免改 1.0 证据脚本语义；触及 CLI/CI/发布文档前 **rebase 最新 main** 并记录 range-diff |

除上表外，依赖选择、ABI v1、packet v1、任务顺序、栈切换规则、证据 ID 与定量门禁 **沿用已批附录，不重开**。

### 1.6 成功标准（产品级）

1. 原生可执行文件完整 title→ending；进程无 WebView/JS/wasm。
2. 同一 demo 在 Web 与 native 上规范化输入/状态 trace 语义对等。
3. 存档/prefs 经 XDG 跨重启恢复；失败可诊断、不静默丢档。
4. 默认 `moonsightc build` 与 `publish-desktop.sh`（Tauri 默认）不回退。
5. 1.1 exact-artifact 证据全 `PASS` 后方可声称 production-supported / coequal。

## 2. 架构

### 2.1 分层剧院模型（不变）

```
*.yuki + assets ──► moonsightc ──► MSB / manifest / catalogs / fonts
                                      │
                    ┌─────────────────┴─────────────────┐
                    ▼                                   ▼
             host_web (wasm)                     host_native (ELF)
             thin export adapter                 MoonBit main + FFI
                    │                                   │
                    └──────────► host_core ◄────────────┘
                                 HostSession (唯一活跃)
                                      │
                         Engine / Stage / UI / GlyphAtlas
                                      │
                    JS WebGPU Host              ms_native_* C ABI
                    (现有 float pack)           SDL / wgpu / FT / ma
```

### 2.2 包与职责

| 单元 | 职责 | 依赖 | 对外接口 |
|------|------|------|----------|
| **`host_core`** | 可测 `HostSession`：boot、tick、输入、UI paint、locale/prefs/save 桥、帧生产、`QueueBackend` | runtime / render / audio / std_* | 目标中立 API；不绑 wasm 或 SDL |
| **`host_web`** | 兼容适配器：现有 `export_*` 名与 float pack | `host_core` + `project_ui` | **现有 TS/wasm ABI 不变** |
| **`host_native`** | 原生 main loop：poll → session → packet/audio/glyph | `host_core` + C stubs | 仅经 gated `extern "c"` 调 `ms_native_*` |
| **`project_ui`** | 顶层共享（自 `host_web/project_ui` 上移） | 无 Host 专属 | moonsightc 临时替换源后恢复 stub |
| **C shim** | 窗口/GPU/字/音/XDG；**不**回调 MoonBit | `native-deps.lock.json` 钉死的上游 | 唯一头文件 `moonsight_native_host.h` |

### 2.3 运行时不变量

1. **单活跃会话** — `HostSession::activate` 在已有所有者时失败（`host-session-active`）；`deactivate` 停逻辑音频、drain、将全局 mixer 置 `NullBackend`。测试可 **顺序** 创建会话证明隔离；**并发活跃会话不在 1.1**。
2. **回调方向** — SDL / wgpu / miniaudio 回调只写有界 POD 队列；**仅 MoonBit 主线程** drain。音频回调禁止分配、文件 I/O、阻塞锁、日志、MoonBit 调用。
3. **FFI 所有权** — 非原始参数一律 `#borrow` / `#owned`；frame packet 与 persistence 字节不跨调用持有；vendor 指针不越过 ABI。
4. **生命周期顺序**  
   - 创建：logging/config → persistence/content 校验 → SDL/window → wgpu → FreeType/fonts/atlas → miniaudio → `HostSession` boot → 帧循环  
   - 销毁：停输入 → flush persistence → 停音频 → GPU idle/释放 → 毁窗/SDL → 释放 C 句柄  
5. **close 语义** — `ms_native_close` 幂等 tombstone；重复 close 成功；其它后续 handle 调用返回 `MS_NATIVE_ERR_CLOSED`。

### 2.4 渲染边界

- 保留 DrawList **语义版本 1** 与现有 sprite/glyph stride（7 / 12）。
- **Web** 继续 float pack export。
- **Native** 使用 `NativeFramePacket` v1：little-endian，`MSFR` 魔数 + 32 字节头 + 与 DrawList v1 相同的 f32 记录。
- 规范 WGSL 位于 `render/shaders/*.wgsl`；Web 以同源字符串导入；native 经确定性生成器嵌入，**源 digest 一致**。
- 符合性测试将 Web float pack 与 native byte packet 解码为同一规范模型后要求语义相等。

### 2.5 每帧数据流

1. C：`ms_native_poll_event` → 项目枚举事件（非 SDL scancode）。
2. MoonBit：解码 → Intent / pointer / wheel / lifecycle。
3. `HostSession` tick → Engine / UI → DrawList 语义。
4. 编码 `MSFR` packet；新 resource / pending glyph 经 define / rasterize-upload。
5. C：`ms_native_submit_frame` + `ms_native_audio_apply` drain。
6. present；设备丢失 → 类型化致命错误（禁止静默回退 demo 或换后端）。

### 2.6 可行性顺序（硬门禁）

```
Task 1  基线锁定（独立 worktree + Web characterization）
   → Task 2  native-only 垂直切片（deps / ABI / SDL / wgpu / ELF / ASan）
        [架构不可行 → 停止；新 RALPLAN 修订；禁止静默换栈]
   → Task 3  host_core 抽取（Task 2 PASS 前禁止改 host_web 生产路径）
   → Task 4–8  渲染 / 字体 / 输入 / 音频 / 持久化
   → Task 9–12 打包 / 符合性 / CI / exact-artifact 证据
```

**Task 2 未 PASS：禁止 `host_core` 抽取与 Web 侧生产行为改动。**

### 2.7 与 Formal 1.0 的边界

- 共享引擎语义（save v5、locale、rollback、MSB/catalogs）**原样复用**。
- 1.1 证据体系 **独立**（native 专用 verifier / ID）；**不复用或改写** W1/D1/C1。
- 存档 **schema 共享、路径不互通**；文档说明迁移指南，**不自动拷贝** Tauri/localStorage 数据。

| 组件 | 版本锚点 | 角色 |
|------|----------|------|
| SDL3 | 3.4.12 | 窗口 / 输入 / surface |
| wgpu-native | 29.0.1.1 | WebGPU + WGSL |
| FreeType | 2.14.3 | 字形栅格 |
| miniaudio | 0.11.25 | 解码 / 混音 / 设备 |
| Noto Sans Regular + Noto Sans CJK SC | 固定 SHA-256（见 PRD） | 打包字体栈 |

锁文件：`host_native/native-deps.lock.json`（URL、commit、archive SHA-256、许可证）。  
HarfBuzz 14.2.1 仅评估，1.1 不链接。

**换栈**仅当 Task 2 确认 PRD 中六类 **架构不可行** 之一，并完成 Planner→Architect→Critic 修订。普通性能、端口 bug、单机驱动异常、可修 wrapper 缺陷 **不** 授权换栈。

备选（仅修订后）：B Sokol；D SDL_GPU；C Dawn（最重，默认拒绝）。

### 3.2 公共 C ABI v1

- 唯一 MoonBit 可见入口：`ms_native_get_abi_version`、`create`、`backend_info`、`poll_event`、`resource_define`、`font_define`、`glyph_rasterize_upload`、`submit_frame`、`audio_apply`、`persistence_read`/`write`/`flush`、`close`。
- 公共结构体以 `struct_size` + `abi_version` 开头；x86_64 固定尺寸（config/event 40B，error 272B）由 `_Static_assert` 锁定。
- 结果码 `0` / `1` / `-1..-15` 与 error category `0..17` 为固定表。
- 项目自有枚举：resource kind、audio command、persistence kind、event type/flags、key code；**禁止** SDL/vendor 枚举泄漏。
- `ms_native_create` 确立 creator 主线程；其后 handle 调用同线程亲和。
- 字符串与输出缓冲：caller-owned；backend-info / persistence-read 两段式 size-query；**缺失** prefs/save = `MS_NATIVE_OK` 且 `out_required=0`（非错误）。
- 私有 `extern "c"` 仅出现在 native-target-gated MoonBit 文件；共享包不得标 native-only。

### 3.3 NativeFramePacket v1

| 字段 | 规则 |
|------|------|
| 头 | 32B LE：`MSFR`、version=1、header_size=32、total_size、sprite/glyph/resource counts、veil_opacity、flags=0 |
| 载荷 | sprite×7 再 glyph×12，f32 LE，语义 = DrawList v1 |
| 上限 | 65 535 sprites、65 535 glyphs、16 MiB 总长 |
| 资源 | 连续不可变前缀；`resource_count` 必须等于已同步前缀 |
| 字形 | 同步 rasterize+upload；成功后 `mark_ready`；**下一引擎帧** 可见 |

### 3.4 字体（跨后端显式迁移）

- 生产路径 **不依赖** 系统字体；Latin 与 CJK 文件与 digest 固定，族名顺序固定。
- Web：加载并校验两文件；去掉 ambient `sans-serif` / 未钉死 `Noto Sans` 回退；未加载字体 → 类型化 `font` 失败。
- **改 Web 前** capture 当前 Chromium / Firefox / Tauri WebKitGTK Latin+CJK 基线。
- 迁移后：既有 Latin 捕获无回归；CJK golden 一次性审查后，四端（三 Web 表面 + native）同一 geometry/mask 门禁。
- FreeType：`FT_LOAD_NO_HINTING | FT_LOAD_NO_BITMAP`、`FT_RENDER_MODE_NORMAL`、gray 256；baseline 公式与现 Web bridge 对齐；失败致命，禁止静默 tofu。
- 跨后端比较：**不要求** Canvas 与 FreeType 抗锯齿像素逐点相同；要求行/reveal/glyph 计数一致、框/基线 ≤1 逻辑像素、mask IoU 等阈值（见 PRD）。

### 3.5 音频

- 继续 `QueueBackend` 作为引擎→Host 逻辑边界。
- 命令序与 Web 一致：play BGM、stop BGM、play SE、set BGM volume。
- 资源仅经 manifest；未知 ID / 解码 / 设备失败可观测。
- Rollback / load 经同一事件流调和逻辑 mixer。

### 3.6 持久化

- 根：`${XDG_DATA_HOME:-$HOME/.local/share}/moonsight/<module-id>/`。
- 文件：`prefs.json`、`saves/<slot>.json`、内部 `.tmp`/`.bak`。
- module-id 路径编码：前缀 `id-`，安全字符保留，其它 UTF-8 字节 `%HH`，禁止截断与空/`.`/`..`。
- 状态机对齐 Tauri：recover → exclusive `.tmp` → write+fsync → last-good→`.bak` → rename → 失败恢复 → parent fsync；故障注入覆盖每步。
- Host 将 JSON 字节视为不透明，直至 runtime 校验；非法/不兼容存档 **不** 静默变异或覆盖。
- **延迟** 跨后端自动迁移；诊断可打印 Tauri appData 模式与只读迁移说明。

### 3.7 构建与打包

| 命令 | 默认 | 扩展 |
|------|------|------|
| `moonsightc build <project> -o <dir>` | **web** | `--backend web \| native-linux-x86_64` |
| `scripts/publish-desktop.sh` | **tauri** | `--backend tauri \| native` → `publish-native.sh` |

- Native 拒绝非 Linux x86_64 构建主机（除非未来显式交叉工具链）。
- 产物：`moonsight-<project>-linux-x86_64.tar.gz`，含 `bin/moonsight-player`、`lib/libwgpu_native.so`、`share/moonsight/<project>/`（内容/字体/着色器/许可证/manifest/MSB）、`build-info.json`。
- RUNPATH：`$ORIGIN/../lib`。SDL3、FreeType、miniaudio 编入项目组件；**仅** `libwgpu_native.so` 为捆绑动态库。
- 发布构建容器：**Ubuntu 22.04 x86_64**；要求符号 **≤ `GLIBC_2.35`**。
- 压缩包 **≤ 150 MiB**；超出须完整共识修订（非 critic-only 例外）。
- 禁止依赖：WebKit、JavaScriptCore、Node、wasm 运行时、`libstdc++`、LLVM 等（详见 PRD ELF 允许列表）。

### 3.8 用户可见错误

启动失败 **非零退出** + 结构化类别：`window-init`、`gpu-adapter`、`gpu-device`、`gpu-surface`、`shader`、`font`、`audio-device`、`content-manifest`、`content-msb`、`resource`、`persistence`、`unsupported-platform` 等。

**禁止：** 静默启动内置 demo、静默换后端、缺内容继续、空渲染器继续。

`--backend-info` 打印：backend、版本、ABI/packet 版本、显示协议、adapter/driver、存储根、module ID、构建 commit。

### 4.2 证据矩阵

#### 自动化（加法）

保留既有 MoonBit / Web / CLI / Tauri / docs 门禁。新增：native contract、release link、sanitizer、package verifier、repro、Xvfb/lavapipe smoke、**`N1-GLIBC235-SMOKE`**（干净 Ubuntu 22.04 安装与动态加载 + 脚本化 title→首句对话）。

#### 真机（mandatory）

| ID | 环境 |
|----|------|
| `N1-WAYLAND-MESA` | Ubuntu 24.04+ LTS，Wayland，Mesa Intel 或 AMD |
| `N1-X11-NVIDIA` | Ubuntu 24.04 或 Fedora，X11，受支持 NVIDIA 专有驱动 |

每条必须绑定 **exact artifact SHA-256** 与 commit；CI 软件渲染 **不能** 替代任一条。

#### 定量门禁（参考机 1920×1080）

- 冷启动至可交互 title：p95 ≤ 5s（10 次）
- 稳态帧间隔：p95 ≤ 18ms，p99 ≤ 34ms（10 分钟）
- 输入至可见：p95 ≤ 50ms
- 30 分钟混合 BGM/SE：无 underrun
- 热身后 30 分钟常驻内存增长 ≤ 32 MiB
- 压缩包 ≤ 150 MiB

`FAIL` / `BLOCKED` / `NOT_RUN` 均阻塞 coequal production 声明。

### 4.4 风险

| 风险 | 应对 |
|------|------|
| wgpu-native ABI/glibc/surface 不可行 | Task 2 硬门禁 + 六类修订规则 |
| `host_core` 抽取回归 Web | characterization 先于抽取；export 名与语义冻结 |
| 双字体改 Web CJK 像素 | 显式迁移 + 四端门禁；Latin 无回归 |
| 误标 production | 证据 schema 先于候选；`NOT_RUN` 阻塞 |
| 与 Formal 1.0 证据纠缠 | 独立 1.1 证据 ID；不改 W1/D1/C1 |
| 包体积 >150 MiB | 完整共识修订 |

### 4.5 验收

仅当同时满足：

1. test-spec 中 A1–F21 全部映射到 `PASS` 检查；  
2. Web/Tauri 回归保持绿；  
3. `N1-WAYLAND-MESA` 与 `N1-X11-NVIDIA` 对 exact artifact 为 `PASS`；  
4. 压缩包、ELF、glibc、无 Web 运行时依赖门禁 `PASS`；  

才可声明 **MoonSight 1.1 native Linux 与 Tauri 对等支持**。
