# MoonSight 1.1 Native Desktop Runtime Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> Also load before any FFI work: `moonbit-c-binding` / `make-moonbit-c-bindings` skill (ownership annotations, stubs, ASan).
>
> Use TDD for each behavior change. Keep implementation in an **isolated git worktree** (create via `superpowers:using-git-worktrees` at execution time).

**Goal:** Ship a production-supported Linux x86_64 native MoonSight desktop backend with no runtime WebView/JavaScript/wasm, coequal with Tauri, consuming the same project/content contract as Web.

**Architecture:** Prove SDL3 + wgpu-native ABI/ELF/surface in a **native-only vertical slice** before touching Web Host production code. Then extract target-neutral `host_core` (`HostSession`, single-active-session around the process-global mixer), keep `host_web` as a thin wasm export adapter, and add `host_native` MoonBit main + project-owned `ms_native_*` C ABI (SDL3 window/input, wgpu-native WGSL render, FreeType glyphs, miniaudio, XDG persistence). Default CLI remains Web; native is explicit.

**Tech Stack:** MoonBit (native + wasm-gc), C stubs (`extern "c"`), SDL 3.4.12, wgpu-native 29.0.1.1, FreeType 2.14.3, miniaudio 0.11.25, WGSL, Python packaging/evidence scripts, GitHub Actions, Xvfb/lavapipe + real Linux GPU evidence.

`docs/superpowers/specs/2026-07-14-moonsight-1.1-native-desktop-design.md`
