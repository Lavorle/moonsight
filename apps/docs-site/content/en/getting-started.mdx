# Getting Started

> **Source of truth:** Engine behavior must match repo docs until migration
> completes; Q2 core pages are authoritative for listed topics.
>
> Distilled from the repo root `README.md` / `README.mbt.md`. When behavior
> diverges, prefer the repo README and package sources until migration is done.

MoonSight is a MoonBit + WebGPU visual novel engine (Phases 1ŌĆō4 + Q1/0.5
playable core + Q2 engine presentation). MoonYuki scripts compile to IR/bytecode,
run on a VM + Stage/Director, and render through a packed draw list consumed by
a JS WebGPU host. System menus and the dialogue HUD use a retained MoonBit UI
tree (`std_ui` + optional project `ui_package`). Desktop uses the same web build
inside a minimal Tauri shell.

## Prerequisites

- [MoonBit](https://www.moonbitlang.com/) toolchain (`moon`)
- A C compiler for native CLI builds (`export CC=gcc` on Linux)
- A **WebGPU-capable** browser (see [Browser / WebGPU](#browser--webgpu-important-on-linux))
- Python 3 (or any static file server) to serve the dist directory

## Quickstart

From the **repository root**:

```bash
export CC=gcc

# typecheck + unit tests
moon check
moon test

# build the Svelte host shell first (moonsightc build needs apps/host-web/dist)
cd apps/host-web && npm i && npm run build && cd ../..
moon build --target wasm-gc --release host_web

# compile sample game + web dist (demo sets ui_package ŌåÆ rebuilds host wasm)
moon run cmd/moonsightc --target native -- build demo/game -o dist/demo

# play in browser (WebGPU; use localhost ŌĆö not file://)
cd dist/demo && python3 -m http.server 8080
# open http://localhost:8080/
```

## Start from an empty project

```bash
export CC=gcc
moon run cmd/moonsightc --target native -- new mygame
cd apps/host-web && npm i && npm run build && cd ../..
moon run cmd/moonsightc --target native -- check mygame
moon run cmd/moonsightc --target native -- build mygame -o dist/mygame
```

`new` copies the monorepo **`templates/minimal`** scaffold (`moonsight.json`,
`main.yuki`, optional `assets/`) into a minimal buildable project. Default UI is
engine `std_ui` (no project `ui/`). You can also pass
`new <name> -o <parent_dir>`. If the destination already exists, the command
fails without overwriting.

## Web host (Svelte)

`moonsightc build` copies the **Svelte** web shell from **`apps/host-web/dist`**
(requires `index.html`). Build that shell first, then package the game.

```bash
export CC=gcc
cd apps/host-web && npm i && npm run build && cd ../..
moon build --target wasm-gc --release host_web
moon run cmd/moonsightc --target native -- build demo/game -o dist/demo
```

With no `ui_package`, moonsightc does not rebuild wasm automatically ŌĆö refresh
into Svelte `public/` for dev when needed:

```bash
moon build --target wasm-gc --release host_web
cp _build/wasm-gc/release/build/host_web/host_web.wasm apps/host-web/public/
```

One-shot packaging of a static web directory: [Build & Publish (Web)](./publish)
(`./scripts/publish-web.sh`).

## Browser / WebGPU (important on Linux)

MoonSight **requires WebGPU**. There is no WebGL fallback.

| Browser | Notes |
|---------|--------|
| **Chrome / Edge / Chromium** | Best path. On **Linux** enable flags if needed (below). |
| **Brave** | Chromium-based; on Linux often needs the same flags as Chrome. |
| **Firefox** | WebGPU shipped more fully on Windows/macOS; **Linux is still experimental** and often needs `about:config` or Nightly. |

### Brave / Chrome (Linux) ŌĆö if you see `WebGPU not available`

1. Open `brave://flags` or `chrome://flags`
2. Enable **Unsafe WebGPU Support** (`#enable-unsafe-webgpu`)
3. Enable **Vulkan** (`#enable-vulkan`) ŌĆö commonly required on Linux
4. Optional: **Ignore GPU blocklist** (`#ignore-gpu-blocklist`)
5. Relaunch, then check `brave://gpu` / `chrome://gpu` (WebGPU should not be ŌĆ£DisabledŌĆØ)

CLI:

```bash
brave-browser --enable-unsafe-webgpu --enable-features=Vulkan --use-angle=vulkan http://localhost:8080/
# or: google-chrome --enable-unsafe-webgpu --enable-features=Vulkan ...
```

### Firefox

1. `about:config` ŌåÆ set `dom.webgpu.enabled` = `true`
2. Also try `gfx.webgpu.force-enabled` and/or `gfx.webgpu.ignore-blocklist` = `true`
3. Restart; if `navigator.gpu` is still missing, use **Firefox Nightly** or a Chromium browser

Always serve via **`http://localhost`** (or https). Opening `file://` blocks WebGPU.

## Play controls (summary)

Cold start is on the **title** modal (Start ŌåÆ entry scene).

| Input | Effect |
|-------|--------|
| Click / Enter / Space / Z | Advance (or activate focused menu button) |
| Esc | System menu |
| ŌåæŌåō / W/S | Focus |
| ŌåÉŌåÆ | Settings sliders |
| H | Backlog |
| Ctrl hold | Skip (burst) |
| 1ŌĆō9 | Select choices |
| A | Auto |
| Ctrl+S / Ctrl+L | Quick save & load slot 0 (`localStorage`) |

Timed `@flow.wait` ignores Advance/skip until the countdown finishes. Menus
pause narrative Advance. Full semantics: [Play input](./play-input).

## Packages (map)

| Path | Role |
|------|------|
| `script` | MoonYuki ŌåÆ IR / `MSB1` (rejects project `- screen`) |
| `runtime` | VM, Director, Stage, UiApp/UiRuntime, prefs, save (v4), tweens + scale |
| `render` | Draw list pack, text layout, kind+z sort, scaleŌåÆsprite size, `UiDrawOp` paint |
| `audio` | Logical BGM/SE mixer (volume / fade) |
| `std_commands` | Standard `@` host commands (layers, dissolve, ui.show/hide, audio) |
| `std_ui` | Default HUD + title / game_menu / save_load / settings / confirm / backlog |
| `host_web` | Browser wasm host (WebGPU entry; shell is `apps/host-web`) |
| `apps/host-web` | Svelte+TS host shell (**required** by `moonsightc`; build `dist/` first) |
| `host_desktop` | Tauri 2 shell |
| `cmd/moonsightc` | `new` / `check` / `build` CLI (scaffold, resource check, optional ui_package) |
| `templates/minimal` | Source tree copied by `moonsightc new` |
| `demo/game` | Sample project (+ optional `ui/` override) |
| `apps/docs-site` | This documentation site |
